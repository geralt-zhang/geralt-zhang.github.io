对于 Vim 的了解最开始是在大学，那时候有个 Linux 的老师让我们看 Git、Vim 这些东西，教学系统是 RedHat 7，我记得当时安装了个 RedHat 8 和 Deepin 来摸索，当时还在 Deepin 里面搞英伟达的驱动，太多 Bug 了，而且全靠官方维护的一个兴趣社区来手动按教程修复。但是那个系统很好看，说来也神奇，我竟然是因为 UI 界面入坑的 Linux，关于 Git 和 Vim 我当时看不懂，也没怎么用过，只觉得这些很难，但也是这种不懂让我对这些神秘的工具感兴趣。

后面第二次接触 Vim 是在读研快毕业，觉得如果要学习 C 和 Linux，用 Vim 总归是好点的，很多极客都是这样，那我可能也需要这样，然后就按照一些教程疯狂的配置插件，在折腾的过程中感觉很繁琐，如果我用 VScode 只需要一个安装包的事，但是 Vim 却要不断折腾，这让我逐渐怀疑极客们的选择，最后让我放弃它的一道坎是我需要用 SSH 来连接，但当时不知道 OSC52 这种剪贴逻辑，和 AI 交流许久也没法配置好 xclip 的复制到本机，无奈放弃。

这是我第三次来尝试 Vim 了，去年进入工作接到的任务让我对 Git 的原理已经熟悉了很多，对 NeoVim 的摸索也在这一年来不断持续，昨天终于耐心读完了 kickstart，并且配置好了 SSH 和本地的剪贴板，我有预感，这次我会用的久一些。
